Motivation

This project introduces N2O Protocols for Haskell programming language. The aim is to be compatible on transport level with Erlang version of N2O. This ecosystem represents a solid CORBA/OASIS/SOAP/XMPP-replacement, ready for high-speed, low-latency IoT, MQTT and WebSocket applications.

N2O Haskell

The philosophy behind N2O is a simplicity, cleanness and purity. The core of N2O shoud be no more than 500 LOC.

The idea to send prerendered JavaScript events over the wire belongs to Rusty Klophaus who made Nitrogen Erlang web framework. Later this was refined by N2O team and now is available in its purity of Haskell.
